1. создается класс "Стол", по шаблону которого потом можно мутить кучу однотипных, но разных столов (разных по номеру, вместимости чел. (размеру)

2. создается класс "Ресторан"
В ресторане можно:
- добавлять столы
- бронировать столы
- освобождать столы
- искать столы на подходящее кол-во человек

Также для классов написаны доп. функции, с помощью которых:
- невозможно добавить, если стол с таким номером уже есть
- невозможно освободить или забронировать стол, если стол с таким номером отсутствует
- невозможно забронировать стол, если стол уже занят

А также еще доп. функции, которые позволяют:
- вывести информацию о столах на складе, чтобы добавить нужные в ресторан
- вывести информацию об уже имеющихся в ресторане столах и их статусах
